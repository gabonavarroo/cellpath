# PHASES.md — 14-day master plan

> Two engineers, 14 calendar days. Read AGENTS.md first to know your role (A or B). Each phase
> lists deliverables, blocking dependencies, success criteria, and fallback plans. Failure to
> hit a phase's success criteria triggers the documented fallback rather than schedule slip
> into the next phase.

## Phase 0 — Day 0: bootstrap (both)

**Goal.** Repo skeleton present, both engineers can run `make setup` and reach a working
`.venv`. Shared interfaces and mock generators are in place so Phase 1 work can start in
parallel.

**Deliverables.**

- [A+B] `make setup` works on Mac and Linux. `python -m src.utils.device` prints `mps` / `cuda` / `cpu`.
- [A+B] `make data` (script exists; may be a no-op shell on Day 0 — actual data download is
  Phase 1 Day 1).
- [A+B] All Hydra configs in `config/` load without error: `python -c "import hydra; ..."` smoke check.
- [A] `src/data/perturbation_pairs.py::generate_mock_pairs()` is implemented and produces
  Contract-2-compliant `.npz` files on disk.
- [B] `src/models/dynamics.py::PerturbationDynamicsModel` skeleton compiles, accepts mock
  inputs of shape `(B, 32)` and `(B,)`, returns the 3-tuple `(z_next, mu, log_var)` with
  correct shapes.
- [A+B] Tests `tests/test_data.py::test_mock_pairs_shape` and
  `tests/test_dynamics.py::test_forward_shape` pass.
- [A+B] First `PROGRESS.md` entry committed.

**Blocking dependencies.** None.

**Success criteria.** `pytest -k "not slow"` exits 0; `make pipeline CONFIG=default --dry-run`
(or its python equivalent) exits 0.

**Fallback if behind schedule.** Skip the docker-cuda build on Day 0 (it is independent of dev
work); roll it into Phase 1.

## Phase 1 — Days 1–3: data + VAE (Agent A) || dynamics architecture (Agent B)

**Goal.** Agent A produces a clean preprocessed AnnData + trained scVI VAE. Agent B implements
dynamics-model architecture and trains on **mock pairs** (Contract 2 schema, generated by
Agent A's `generate_mock_pairs`). Real-data dynamics training is Phase 2.

### Agent A deliverables

- [A] `src/data/download.py::download_norman()` — pertpy `dt.norman_2019()` primary path, GEO
  fallback. Stores into `data/raw/`.
- [A] `src/data/preprocess.py::run_preprocessing(cfg)` — full pipeline:
  - Preserve raw counts in `adata.layers["counts"]`.
  - `sc.pp.filter_cells(adata, min_counts=cfg.data.min_counts)`.
  - `sc.pp.filter_genes(adata, min_cells=cfg.data.min_cells)`.
  - HVG selection via `sc.pp.highly_variable_genes(adata, flavor="seurat_v3",
    n_top_genes=cfg.data.n_hvg, layer="counts")`.
  - `sc.pp.normalize_total(adata, target_sum=1e4)` and `sc.pp.log1p(adata)` —
    operates on `adata.X`, leaves `adata.layers["counts"]` untouched.
  - Subset to HVG and assign integer `perturbation_idx`.
  - Save `data/processed/norman_hvg.h5ad`.
- [A] `src/models/vae.py::train_vae(cfg, adata)`:
  - `scvi.model.SCVI.setup_anndata(adata, layer="counts")` — **counts layer**.
  - `model = SCVI(adata, n_latent=32, gene_likelihood="nb", n_layers=2, dropout_rate=0.1)`.
  - `model.train(max_epochs=cfg.vae.max_epochs, early_stopping=True)`.
  - `model.save("artifacts/vae/model/", overwrite=True)`.
  - Write `latents.h5ad`, `gene_vocab.json`, `z_reference_centroid.npy`, `epsilon_success.json`.
- [A] `scripts/train_vae.py` orchestrates the above with Hydra.

### Agent B deliverables

- [B] `src/models/dynamics.py::PerturbationDynamicsModel`:
  - Constructor: `n_latent`, `n_genes`, `d_emb`, `n_hidden`, `n_layers`, `log_var_min/max`.
  - `gene_embedding = nn.Embedding(n_genes + 1, d_emb)` (+1 = NO-OP, unused at train time).
  - MLP trunk with residual blocks, SiLU activations.
  - Two heads: `mu` (linear → 32), `log_var` (linear → 32, clamped).
  - `forward(z, gene_idx) -> (z_next, mu, log_var)`.
- [B] `src/models/embeddings.py::GeneEmbedding` helper (in case Agent B wants to swap to a
  factorized embedding later for OOD generalization).
- [B] `scripts/train_dynamics.py`:
  - Hydra-driven. Reads `artifacts/pairs/train_pairs.npz` (real or mock).
  - Heteroscedastic Gaussian NLL loss; composition loss for combos when present.
  - TensorBoard logging from epoch 1.
  - Saves to `artifacts/dynamics/model.pt` + `config.json`.

### Blocking dependencies

- B's dynamics training depends on **either** A's real pairs (Phase 2) **or** A's `generate_mock_pairs`
  (Phase 0). Day 1 onwards B uses mocks; Day 4 onwards B switches to real pairs.

### Success criteria

| Criterion | Value | Owner |
|---|---|---|
| Preprocessing runs without error | exit 0, h5ad file > 100 MB | A |
| HVG count | exactly `cfg.data.n_hvg` (default 2000) | A |
| scVI ELBO converged | early stop triggered or epochs > 200 with negative-ELBO trend | A |
| Latent UMAP visual | control vs perturbed visibly separated | A |
| ε_success computed | written to disk, value < 5.0 (sanity) | A |
| Dynamics shapes | `(B,32), (B,), (B,32), (B,32), (B,32)` | B |
| Dynamics smoke train on mock | 100 steps; loss decreasing | B |

### Fallback if behind schedule

- If preprocessing balks on Norman (e.g. pertpy cache fails), fall back to the published
  scperturb.org `.h5ad` URL (documented in DATA.md). The GEO MTX fallback is **not** time-
  budgeted for Phase 1 — only attempted if both pertpy and scperturb fail.
- If scVI fails to converge, reduce `n_latent` to 16 and `n_hidden` to 128. The smaller model
  is faster and almost always converges.

## Phase 2 — Days 4–6: latent validation (A) || dynamics training + gate (B)

**Goal.** Agent A validates the VAE latent's biological sensibility and produces the real
OT-paired training data. Agent B retrains dynamics on real pairs and **passes the primary
validation gate**. RL training is gated on this.

### Agent A deliverables

- [A] `src/data/perturbation_pairs.py::build_pairs(cfg, adata, Z, method)`:
  - OT path: per-perturbation Sinkhorn with `reg=cfg.pairing.ot_epsilon`.
  - Cost matrix: `||z_ctrl_i - z_pert_j||_2`, median-normalized.
  - Sample hard pairing once per epoch (writer of pairs writes once; consumer re-samples
    in-memory if requested).
  - Random fallback + mean-delta fallback selectable via `cfg.pairing.method`.
  - 90/10 within-perturbation cell split → `train_pairs.npz` / `val_pairs.npz`.
  - 80/20 perturbation split → `ood_pairs.npz` (the 20% held-out perturbations).
  - Combos: 80% → `combo_pairs.npz`; 20% held out for composition-generalization eval.
- [A] `src/analysis/latent_space.py` — UMAP, silhouette score, ARI, centroid distance plots.
- [A] `notebooks/02_vae_latent_inspection.ipynb` final figures.

### Agent B deliverables

- [B] `src/analysis/metrics.py::dynamics_validation_gate(...)` — implements the gate per
  AGENTS.md §1 Phase 2.
- [B] `scripts/train_dynamics.py` runs the gate at end of training and writes
  `artifacts/dynamics/gate.json`.
- [B] Document any baseline that wins by reverse-engineering why; this is a *learning signal*
  about the data and is NOT cause to lower the gate threshold.

### Validation gate (recap)

Primary gate (must pass to unlock Phase 3 RL):

| Baseline | Metric | Required margin |
|---|---|---|
| No-op (Δz = 0) | held-out cell R² | MLP > no-op by ≥ 0.10 |
| Global mean-Δ | held-out cell R² | MLP > global mean-Δ by ≥ 0.05 |
| Per-gene mean-Δ | held-out cell R² | MLP ≥ per-gene mean-Δ (no regression) |
| Linear ridge per gene | held-out Pearson R | MLP > by ≥ 0.03 |
| Nearest-neighbor (k=5) | held-out R² | MLP > by ≥ 0.03 |
| Uncertainty calibration | Spearman(σ², (Δz - Δẑ)²) | ≥ 0.20 |

OOD report (NOT gating, reported only): same metrics on `ood_pairs.npz`. If OOD R² < 0,
document but proceed.

### Blocking dependencies

- B's gate depends on A's real `train_pairs.npz` (Day 4).
- A's UMAP / silhouette work depends on her own VAE output (Phase 1 complete).

### Success criteria

| Criterion | Value | Owner |
|---|---|---|
| OT Sinkhorn converges | no NaN, transport plan non-degenerate | A |
| ≥3 pairing methods runnable | switch via Hydra `pairing.method` | A |
| Silhouette on perturbation | Informational — not a hard gate (see note below) | A |
| Primary validation gate | passed | B |
| OOD report | written, regardless of pass/fail | B |

**Note on silhouette (updated 2026-05-13):** The original threshold ≥ 0.05 was overly optimistic
for vanilla unsupervised scVI. Empirical result on Norman 2019: silhouette = −0.059 across all
237 perturbation labels. This is expected and documented in the scVI literature for three reasons:
(1) scVI's ELBO optimizes NB reconstruction + KL divergence — it has zero perturbation-label
supervision, so it does not learn to separate perturbation clusters; (2) the KL regularization
forces q(z|x) → N(0, I), which actively smooths the latent and reduces inter-cluster separation;
(3) with 237 biologically overlapping CRISPRa perturbations, many clusters share similar latent
positions by biology, not by model failure. The latent is confirmed biologically meaningful by
the metrics that actually matter for RL: ε_success = 4.52 ∈ (0.1, 10) ✓, separation ratio
(perturbed / ctrl intra-centroid distance) = 1.146 ✓, and 33.6% of perturbed cells lie beyond
ε_success ✓. Silhouette remains reported as an informational diagnostic in
`artifacts/vae/latent_quality.json`; it is not a Phase 2 gate. If a future version requires
stronger cluster separation, the fix is SCANVI (semi-supervised, uses perturbation labels) — not
lowering or redefining the silhouette formula.

### Fallback if behind schedule

- If OT diverges or is too slow, switch the **default** pairing method to `mean_delta` for v1
  and run `ot` as an ablation in Phase 4. Document in EXPERIMENTS.md.
- If the gate fails, escalate: examine VAE (does it actually separate perturbations?), examine
  pair quality (does `mean_delta` baseline beat the random pairer baseline? if not, pairing
  itself is broken). Do **not** lower the gate criteria. Failing the gate by Day 6 EOD means
  Phase 3 is rescoped to use mean-Δ-based dynamics with a recorded limitation in the thesis.

## Phase 3 — Days 7–9: analysis (A) || RL environment + PPO (B)

**Goal.** Agent A builds DepMap enrichment and trajectory analysis tools. Agent B implements
the RL env, trains MaskablePPO, produces rollouts.

### Agent A deliverables

- [A] `src/analysis/depmap_validation.py` — hypergeometric, GSEA preranked, null comparison.
- [A] Data: K562 Chronos table from DepMap (download in `scripts/download_data.sh`).
- [A] `src/analysis/trajectory.py` — UMAP projection of rollouts, faceted plots.
- [A] `notebooks/01_data_exploration.ipynb` final figures (perturbation count distributions,
  HVG distributions, QC stats).

### Agent B deliverables

- [B] `src/rl/environment.py::CellReprogrammingEnv`:
  - `gymnasium.Env` subclass.
  - `observation_space = Box(...)` shape `(32,)`.
  - `action_space = Discrete(n_genes + 1)`, where `n_genes` is loaded from
    `artifacts/vae/gene_vocab.json::n_genes` and `+1` is the NO-OP / terminate action.
  - `reset(seed=None, options=None) -> (obs, info)` with `info["action_mask"]`.
  - `step(action)`:
    - If `action == noop_idx`: `terminated = True`; `success = (||z - z_ref|| < ε)`;
      reward = current shaping − any episode-final bonus/penalty.
    - Else: pass through dynamics; update repeat-mask; recompute success.
    - Truncation at step `K`.
- [B] `src/rl/reward.py` — composable reward: distance + sparsity + uncertainty.
- [B] `src/rl/train_ppo.py` — `sb3_contrib.MaskablePPO`.
- [B] `scripts/train_rl.py` reads `gate.json` and refuses unless `passed=true` (override flag
  exists but logs a P0 warning).
- [B] Tests in `tests/test_environment.py` cover gymnasium API compliance, NO-OP success vs
  not-success distinction, repeat-mask correctness.

### Blocking dependencies

- B's RL training depends on B's own gated dynamics (Phase 2) + A's VAE artifacts (Phase 1).
- A's DepMap analysis depends on at least one set of RL rollouts; if RL is delayed, A may
  use *untrained-policy* rollouts as a smoke baseline (clearly labelled).

### Success criteria

| Criterion | Value | Owner |
|---|---|---|
| DepMap enrichment | at least one FDR-adjusted q < 0.05 finding | A |
| Env gymnasium compliance | `gymnasium.utils.env_checker.check_env(env)` passes | B |
| NO-OP semantics correct | unit test: NO-OP at distance > ε ⇒ success=False | B |
| Repeat-mask correct | unit test: action used in step 1 is masked at step 2 | B |
| PPO trains | 2e6 timesteps without crash; success-rate curve trends up | B |

### Fallback if behind schedule

- If PPO training is unstable, reduce `n_steps` and `batch_size`, lower `ent_coef` floor; if
  still unstable, switch to vanilla discrete PPO (without masking) and add a soft repeat
  penalty in the reward.
- If DepMap enrichment shows no signal, expand the gene-set library (add more MSigDB collections),
  and document negative result honestly.

Scoped in [`V3_RESEARCH_PLAN.md`](V3_RESEARCH_PLAN.md). Central hypothesis: a higher-dim or

## Cross-phase invariants

- **PROGRESS.md is updated at the end of every work session.** Phase work without progress
  notes is not considered complete.
- **No phase advances without its success criteria met.** If a fallback is invoked, the
  criterion is the *fallback* criterion, not the original — documented in PROGRESS.md.
- **The validation gate is sacred.** Phase 3 RL work does not start until Phase 2's gate
  passes. The override flag is documented in code but its use must be justified in PROGRESS.md.
- **All work runs through Hydra.** No engineer hand-edits a `.yaml` file at runtime. New configs
  go in `config/experiments/`.
